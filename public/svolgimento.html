Realizzazione

Per realizzare concretamente il database e passare quindi da un modello concettuale a logico, e, successivamente, ad un DBMS si procede come prima cosa redigendo la lista delle entità con i realtivi attributi.
Successivamente si individuano le relazioni tra le entitá e si continua applicando le regole di derivazione. Infine si crea la struttura E/R individuata in un DBMS, come MySQL, in modo da poterci lavorare.

Entità

  • Tutor Scolastico: ( {CodTutSc}, Nome, Cognome, CodFisc )
  • Specializzazione: ( {CodSpec}, Nome )
  • Classe: ( {CodClas}, NomeClasse, AnnoScolastico, FK_TutSc, FK_Spec )
  • Alunno: ( {CodAlu}, Nome, Cognome, CodFisc, DataNasc, FK_classe )
  • Tirocinio: ( {CodTir}, Inizio, Fine, TotOre, Descr, ValTest, ValVoto, FK_alu, FK_azienda )
  • Diario: ( {CodDia}, Data, TipoAtt, Descr, Ore, FK_tirocinio )
  • Azienda: ( {CodAz}, Nome, PIVA, NomeRap, SedeLegale, SedeTirocinio, Telefono, Email )
  • Tutor Aziendale: ( {CodTutAz}, Nome, Cognome, CodFisc, Telefono, Email, DataNasc )

Sono poi stati aggiunti degli attributi aggiuntivi nel DB per implementare delle funzioni al futuro sito di inserimento/visualizzazione dati.

In particolare, per l'autenticazione degli utenti:

  • Alunno: ( Email, Password )
  • Tutor Scolastico: ( Email, Password )

Per la visualizzazione della Mappa interattiva delle aziende:
  • Azienda: ( Latitudine, Longitudine )

Relazioni

  • Tutor Scolastico - Classe: 1 : N
  • Specializzazione - Classe: 1 : N

  • Classe - Alunno: 1 : N

  • Alunno - Tirocinio: 1 : N

  • Tirocinio - Diario: 1 : N
  • Tirocinio - Azienda: N : 1

  • Azienda - Tutor Aziendale: 1 : N

Regole di Lettura

  • Una o più classi appartengono ad una specializzazione
  • Una specializzazione ha una o più classi
  • Ogni classe ha un Tutor Scolastico
  • Ogni Tutor Scolastico puó gestire una o più classi

  • Ogni classe è formata da più alunni
  • Ogni alunno fa parte di una classe

  • Ogni alunno può svolgere più tirocini
  • Ogni tirocinio viene svolto da un solo alunno

  • Ogni tirocinio contiente più diari
  • Un diario apartiene ad un solo tirocinio

  • Un tirocinio viene svolto in una sola azienda
  • Ogni azienda può ospitare più tirocini

  • Ogni azienda ha uno o più Tutor aziendali
  • Ogni Tutor aziendale lavora in una sola azienda

Query

Per verificare il corretto funzionamento della struttura del Database si effettueranno 2 query SQL sul db.
Verranno riportati sotto al codice i risultati delle stesse.

Query 1

Selezionare tutti gli studenti che hanno effettuato lo Stage sotto la supervisione del tutor aziendale "Alessandro Ocagli"

SELECT al.Cognome, al.Nome
FROM alunno AS al, tirocinio AS t, tutor_aziendale AS ta
WHERE ta.Nome = "Alessandro" AND ta.Cognome = "Ocagli" AND ta.FKAz = t.FKAz AND t.FKAlu = al.CodAlu
GROUP BY al.Nome, al.Cognome
ORDER BY al.Cognome

Query 2

Selezionare l'azienda che ha ospitato più tirocini

SELECT a.Nome, MAX(x.val) AS numTirocini
FROM (
SELECT COUNT(*) AS val
FROM azienda AS a, tirocinio AS t
WHERE t.FKAz = a.CodAz
GROUP BY a.CodAz
) AS x, azienda AS a
